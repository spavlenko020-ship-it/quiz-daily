## Things we previously got wrong (and corrected)
- Pluto listed as smallest planet — Pluto was reclassified in 2006. Replaced distractor with Saturn.
- Sahara ambiguity — "largest desert" is Antarctica (ice). Changed question to "largest HOT desert", distractors adjusted.
- Questions file was declared multilingual in one prompt while the code still stored flat English strings — caused runtime `[object Object]` when screens tried to render. Lesson: structural changes to `questions.js` must migrate the Quiz class's `_localized()` helper in the same commit.
- Native `navigator.share` on Windows desktop shows the system picker (Outlook, Teams) — useless for viral sharing. Mobile-only gate added via `isMobile()` UA check; desktop always falls back to clipboard.
- Facebook share button was removed because FB reduces reach on posts that contain explicit share intents (the "share sheet" penalty). Telegram/WhatsApp/Viber + clipboard are the supported routes.
- Wordle is NOT our reference. Wordle is a web-native once-a-day puzzle. Our primary platform is Facebook Instant Games, where the monetization model is high session count × many sessions per day. Reference games going forward: AHA Quiz (16M MAU), WOW Quiz (8.3M), Nametests (6.8M). Every UX decision must be evaluated against "would AHA Quiz do this?"
- FB Instant Games re-engagement patterns that are mandatory for this project: (1) auto-continue countdown on result screen — "Next game in 3..2..1" with a big primary KEEP PLAYING button that's visually 2x larger than the Back button; (2) intra-session combo system with escalating labels (3 in a row = ON FIRE, 5 = UNSTOPPABLE, 7 = GOD MODE) and score multipliers; (3) progress bars everywhere ("You're X away from your best", "Top X% today"); (4) never demotivate — no copy like "does not count", "just practice", "unranked"; every alternative mode must have its OWN hook ("BEAT YOUR SCORE", "DAILY RECORD", "SPEED RUN").
- Question count per session is 12 (not 10). Do not hardcode the number anywhere in UI copy — always read from the config constant so future changes are one line.
- Auto-continue countdown on result screen was MY (the architect's) hallucination. Verified: Trivia Crack has no auto-continue (uses lives system that STOPS players). Hyper-casual games (Flappy Bird, Stack, Helix Jump) have manual tap-to-restart. No top game auto-starts next session. Removed.
- "Beat Your Score" mechanic has a GLASS CEILING (max = best score, then motivation dies). Replaced with LEVEL + XP (infinite progression like Trivia Crack's 240+ levels) and Coins (soft currency that depletes — will gate skins/boosts in Stage 6+).
- Question pool had 3 difficulty tiers but selection was random across them. Fixed: progressive curve now mandatory — easy first (warm-up), medium middle, hard finale. Matches HQ Trivia / Who Wants to be a Millionaire. Hard questions award 1.5× points so the finale feels cinematic.
